GitHub Actions のキャッシュ戦略を徹底的に最適化する

CI/CD Performance GitHub Actions
R
Rym
2026-03-15 2026-03-20 8 min read

はじめに

GitHub Actions のビルドが遅い — これは多くのチームが抱える課題です。本記事では、キャッシュ戦略の最適化によりビルド時間を70%短縮した実践的なアプローチを紹介します。

キャッシュの基本戦略

actions/cache の適切な設定は、CI/CDパイプラインのパフォーマンスに直結します。キーの設計とリストア戦略がポイントです。

.github/workflows/ci.yml- uses: actions/cache@v4
  with:
    path: |
      node_modules
      .next/cache
    key: ${{ runner.os }}-deps-${{ hashFiles('bun.lock') }}
    restore-keys: |
      ${{ runner.os }}-deps-

並列実行の最適化

テスト分割と matrix ストラテジーを組み合わせることで、さらなる高速化が可能です。

計測結果

最適化前後で依存関係のキャッシュヒット率が大幅に向上し、コールドスタートの発生頻度が減少しました。

まとめ

キャッシュ戦略の最適化は、開発チーム全体の生産性に直結する重要な投資です。

v7 — Color & Responsive

v7: Light Mode — Neutral White

  • --bg-primary: #fafcfb → #fafafa (ニュートラル白)
  • --bg-secondary: #f1f5f3 → #f4f4f5 (zinc-100)
  • --bg-code: #f4f7f5 → #f4f4f5
  • --hero-bg: #f0fdf4 → #f0fdf8 (Hero だけ薄緑tint維持)
  • グリーンはアクセント要素のみに限定

v7: Dark Mode — Teal Shift

  • --accent: #34d399 → #5eead4 (teal-300, 低彩度)
  • --accent-2: #6ee7b7 → #99f6e4 (teal-200)
  • --tag-text: #6ee7b7 → #99f6e4
  • --accent-gradient: #0d9488 → #5eead4
  • glow/shadow/dot 全て teal ベース に統一
  • ターミナル内テキスト色も teal に統一
  • 効果: 彩度↓ + 青み↑ = 目に優しく、長時間閲覧に耐える

v7: Responsive 強化

  • 1024px: グリッド minmax 調整、nav gap 縮小、section padding 縮小
  • 768px: padding 24px→16px、About 中央揃え、nav 縮小、control bar 圧縮
  • 480px: hero h1 28px、nav 12px gap、list-card thumb 80px、toolbox 1列
  • フッター: モバイルで中央揃え + アイコン中央配置
